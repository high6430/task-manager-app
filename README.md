## 📋 プロジェクト概要

IT専門学校の卒業制作として開発した、個人のワークスタイルに合わせてカスタマイズできるタスク管理モバイルアプリケーション「PikaDo」です。

### 開発背景

既存のタスク管理アプリでは、以下の課題がありました。機能が多すぎて使いこなせない、特定の用途に特化しすぎている、自分の管理スタイルに合わせられないといった問題がありました。

これらの課題を解決するため、「シンプルながら柔軟にカスタマイズでき、自分だけの使い方ができるタスク管理アプリ」を開発しました。

---

## ✨ 主な機能

### タスク管理

タスク管理機能では、タイトル、締切日時、優先度、詳細を設定してタスクを追加できます。既存タスクの修正・削除も可能で、削除時には確認ダイアログが表示されます。タスクは未対応、進行中、完了の3段階で管理でき、ボタン操作で簡単にステータスを変更できます。また、タスクカードを長押しすることで詳細情報を確認することも可能です。

### 通知機能

設定した時刻に締切通知を送信します。カスタム通知タイミング機能では、「前日20時」「当日朝8時」など自由に通知セットを作成・管理できます。通知から直接タスクを完了・詳細確認することも可能です。アプリ全体の通知をON/OFFで一括制御できるほか、Android 13に対応したSCHEDULE_EXACT_ALARM権限、Android 12以上の正確な時刻通知に対応しています。

### ラベル管理

名前と色を選択してカスタムラベルを作成できます。1つのタスクに複数のラベルを付与することができ、ラベルでタスクを絞り込み表示できます。初期ラベルとして仕事（青）、私生活（緑）、緊急（赤）が用意されており、自分の管理スタイルに合わせてラベルを追加・カスタマイズできます。

### 視覚的な管理

優先度は高（赤）、中（オレンジ）、低（緑）で色分けされます。締切の色分けでは、期限切れが赤、本日締切・時刻前がオレンジ、本日締切・時刻後が赤、余裕ありが緑で表示されます。未対応・進行中・完了をタブで切り替え表示でき、色付きチップでラベルが視覚的に表示されます。

### データ管理

SharedPreferencesによるローカル保存で、ネット接続不要でオフライン動作が可能です。タスクは自動的に締切順に整列されます。

## 🚀 トラブルシューティング

### 通知が届かない場合

**権限設定を確認してください**。通知権限の確認は「設定 → アプリ → PikaDo → 通知」で「通知を許可する」がONになっているか確認してください。アラーム権限の確認は「設定 → アプリ → PikaDo → 権限 → アラームとリマインダー」で有効化されているか確認してください。

**バッテリー最適化を確認してください**。「設定 → バッテリー → バッテリー最適化」で PikaDo を「最適化しない」に設定してください。

**バックグラウンド実行を許可してください**。「設定 → アプリ → PikaDo → バッテリー」で「バックグラウンドでの実行」を許可してください。

**省電力モードをOFFにしてください**。通知が遅延する可能性があります。

### ダウンロードエラーが出た場合

ブラウザのキャッシュをクリア、別のWi-Fi/モバイルデータで試す、ファイアウォールの設定を確認してください。

### インストール後、アプリが起動しない場合

端末を再起動、Google Play ストアから「Google Play開発者サービス」を更新、アプリをアンインストール後、再度インストールしてください。

## 📂 プロジェクト構成

本プロジェクトは保守性と可読性を考慮し、機能ごとにファイルを分離しています。

```
lib/
├── main.dart                                  # エントリーポイント
├── models/                                    # データモデル層
│   ├── task.dart                             # Taskクラス、Priority enum
│   ├── label.dart                            # Labelクラス
│   ├── notification_timing.dart              # 通知タイミングモデル
│   ├── notification_set.dart                 # 通知セットモデル
│   └── app_settings.dart                     # アプリ設定モデル
├── services/                                  # ビジネスロジック層
│   ├── task_service.dart                     # タスク管理
│   ├── label_service.dart                    # ラベル管理
│   ├── notification_service.dart             # 通知管理
│   ├── notification_set_service.dart         # 通知セット管理
│   └── app_settings_service.dart             # アプリ設定管理
├── screens/                                   # 画面層
│   ├── task_board_screen.dart                # タスクボード画面（メイン）
│   ├── task_detail_screen.dart               # タスク詳細画面
│   ├── label_settings_screen.dart            # ラベル管理画面
│   ├── notification_set_settings_screen.dart # 通知セット管理画面
│   └── settings_screen.dart                  # アプリ設定画面
├── widgets/                                   # 再利用可能なUI部品
│   ├── task_card.dart                        # タスクカード表示
│   ├── add_task_dialog.dart                  # タスク追加ダイアログ
│   ├── edit_task_dialog.dart                 # タスク編集ダイアログ
│   └── label_chip.dart                       # ラベル表示チップ
└── utils/                                     # ユーティリティ
    └── logger.dart                           # カスタムログ
```

### 設計方針

各ファイルが1つの明確な役割を持つ単一責任の原則に従っています。データ、ロジック、UIは分離されており、ウィジェットは独立させて他の画面でも使用可能にしています。各層が独立しているため、単体テストが容易です。パフォーマンス最適化では、StatelessWidget化による描画効率向上、フィルタリング処理のキャッシュ化、不要なSharedPreferences読み込みの削減を実施しています。

## 📱 使い方

### タスクの追加

右下の「+」ボタンをタップしてください。タイトル、詳細、締切日時、優先度を入力してください。ラベルを選択します（複数選択可能）。通知を設定してください（オプション）。「追加」ボタンをタップします。

### タスクの編集

タスクカードの「編集」ボタンをタップしてください。内容を修正してください。「保存」ボタンをタップします。

### タスクの詳細表示

タスクカードを長押ししてください。詳細画面が表示されます。編集・完了・削除が可能です。

### タスクの移動

「進行中へ」ボタンで未対応から進行中へ移動、「完了へ」ボタンで進行中から完了へ移動、「未対応へ」ボタンで進行中から未対応へ戻すことができます。

### タスクの削除

タスクカードの「削除」ボタンをタップしてください。確認ダイアログで「削除」を選択します。

### ラベル管理

右上のラベルアイコンをタップしてください。「+」ボタンで新しいラベルを追加できます。ラベル名と色を選択します。削除する場合は各ラベルの削除アイコンをタップします。

### ラベルフィルタ

タブの上にあるフィルタドロップダウンをタップしてください。表示したいラベルを選択します。「すべて」を選択すると全タスクが表示されます。

### 通知セットの管理

右上の設定アイコンをタップしてください。「通知セット管理」を選択します。「+」ボタンで新しい通知セットを作成できます。名前と通知タイミングを設定します。タスク追加・編集時に作成した通知セットを選択可能です。

### アプリ設定

右上の設定アイコンをタップしてください。「通知を有効にする」をON/OFFで一括制御できます。通知権限・アラーム権限の状態を確認できます。

**名前**: 田中 健太（仮名）、**年齢**: 28歳、**職業**: フリーランスWebデザイナー

### 背景と課題

複数のクライアント案件を同時進行しながら、個人プロジェクトやスキルアップのための学習も並行して進めています。プライベートでは趣味のイラスト制作やジム通いも継続したいと考えています。仕事、学習、趣味、プライベートのタスクが混在し、優先順位がつけづらいです。既存のタスク管理アプリは機能が多すぎて使いこなせず、自分のワークスタイルに合わせてカスタマイズしたいというニーズがあります。

### ニーズと選択理由

スマホで簡単にタスクを追加・確認・完了できることが必要です。自分なりのラベルで分類し、必要なタスクだけを表示したいと考えています。締切を忘れないように通知してほしく、シンプルだけど必要な機能は揃っているアプリが欲しいと思っています。

PikaDoを選ぶ理由は、自分の管理スタイルに合わせてラベルをカスタマイズできること、通知タイミングも自由に設定できることです。シンプルな操作で直感的に使える、オフラインでも使えるので場所を選ばないという点も魅力的です。
